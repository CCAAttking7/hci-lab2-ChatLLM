# 项目功能与布局状态追踪

本文档用于记录和追踪当前大模型交互实验平台（HCI-Lab2-LLM）的功能点、UI布局和迭代历史，以便后续维护与更新。

## 当前核心功能
- **前端/后端解耦架构**：
  - `app.py`: 应用入口。
  - `llm_lab/controller.py`: 业务逻辑控制，包括对话提交、切换会话等。
  - `llm_lab/state.py`: 会话状态管理。
  - `llm_lab/ui.py`: Gradio 界面组件渲染与事件绑定。
  - `llm_lab/styles.py`: 自定义 CSS 样式。
- **多模型支持**：DeepSeek-R1 (仅文本)、Qwen3-Max (仅文本)、Qwen3.5-Plus (支持多模态图文输入)。
- **双工作模式**：
  - **标准模式**：单模型对话，支持流式输出和图片上传。针对不支持图片的模型，点击上传会自动拦截并进行提示。
  - **对比模式**：选择两个模型对同一 Prompt 进行双排同步流式对比输出。
- **历史记录管理**：
  - 左侧边栏展示历史会话列表，最多保留 24 条记录。
  - 支持“新建对话”、“切换对话”、“删除对话”、“清空当前对话”。
- **资源统计**：左下角实时显示当前会话的 Token 使用量与上下文占比。

## UI 布局结构
1. **全局布局**：采用左右分栏的卡片式设计，整体带背景模糊和柔和阴影，具有现代感（类似 ChatGPT / Gemini）。
2. **左侧边栏 (`#sidebar-shell`)**：
   - 顶部：“新建对话”按钮（紧凑设计）。
   - 中部：历史记录滚动列表，当前选中的会话高亮显示，悬浮时右侧显示删除图标。
   - 底部：操作区（清空当前、Temperature 调节）和统计信息卡片（Token、上下文）。
3. **右侧主内容区 (`#main-shell`)**：
   - 顶部：模式切换 Tabs（标准模式 / 对比模式）。
   - 中部：聊天面板（Chatbot）。
   - 底部：多模态输入框（支持图片、文本输入）和发送按钮。

## 迭代历史记录

### [最近更新]
- **输入框统一**：将所有模型的输入框统一为 `MultimodalTextbox`，非多模态模型采用前端 JS 拦截点击上传事件并自动提示。
- **发送即清空**：用户点击发送或回车后，输入框内容瞬间清空，无缝衔接下一次输入。
- **会话切换逻辑修复**：新建对话自动插入列表最上方；修复了因返回值数量不匹配导致的 `ValueError` 报错。
- **UI 与交互精修 (已完成)**：
  - **修复左侧栏UI留白过大**：由于Gradio组件自带包裹层的边距和背景色，导致“新建对话”和标题存在巨大的白色背景块和多余的上下边距。目前已通过更深层的 CSS 透传彻底消除了这部分不必要的边距和背景。
  - 优化左侧栏布局，修复底部 Token 和上下文统计被挤出视口的问题。
  - **拆分 Token 与上下文统计**：响应反馈，将 Token 使用量（计费依据）与上下文占用（模型长度限制）拆分为两个独立的显示卡片，并且上下文统计会实时更新百分比。
  - **提升 Temperature=0 的一致性**：在底层 `model_provider.py` 中增加了对 Temperature=0 时的 `seed` 参数注入，尽可能保证生成结果的确定性（注：受底层架构如稀疏 MoE 的影响，完全确定性在部分模型中无法 100% 保证）。
  - **无缝切换单聊与对比模式**：当用户在上方 Tab 切换“标准模式”和“对比模式”时，系统会自动寻找最近的一个对应模式的历史会话并恢复；如果对应模式没有历史记录，则自动创建一个新的对应模式会话，保证两种模式的记录互不干扰。
  - 缩减历史记录项之间的间距，采用类似 Gemini 的极简间距和高亮效果。
  - 对比模式输入框已替换为与标准模式一致的多模态输入框，自带发送图标。
  - 实现了打断模型生成功能：在生成过程中点击其他历史对话，立即打断并无缝跳转。
  - 修复模型下拉选择框高度被拉伸导致弹出层距离原组件过远的问题，现在下拉列表会紧贴在选择框下方。
- **代码健壮性自查与 Bug 修复 (已完成)**：
  - **修复多模态历史崩溃问题**：当用户在 Qwen-Plus 中发送图片后切换到不支持图片的模型（如 DeepSeek-R1），由于历史记录包含了 `image_url` 节点，会导致 DeepSeek API 直接崩溃。已在 `model_provider.py` 中增加了深拷贝和节点清理机制，自动过滤不兼容模型历史中的图片，防止应用崩溃。
  - **修复前端聊天框图片丢失**：之前的逻辑在将消息追加到聊天界面（`base_history`）时，由于格式问题丢失了图片的显示。现已恢复 OpenAI 的标准结构 `[{"type": "image_url", ...}, {"type": "text", ...}]`，确保用户上传的图片能在界面上正确渲染展示。
  - **修复嵌套列表格式错误**：修复了 `model_provider.py` 在处理多模态组装时可能将 `"text"` 字段错误地赋为列表的隐患，保证完全遵守 OpenAI 官方协议结构。
  - **修复发送瞬间的 Token 滞后更新**：点击发送时的第一帧（用于清空输入框和显示用户提问）所携带的 Token HTML 统计是上一次对话的。已在发送瞬间提前计算并下发最新的统计 HTML，消除了界面滞后感。
  - **增强文件路径提取的兼容性**：Gradio 在不同版本和上传场景下，图片返回对象可能是字典、字符串或特定的 FileData 类。重写了 `state.py` 中的 `normalize_image_path`，完美兼容所有的提取场景。
